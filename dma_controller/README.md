# High-Speed Data Transfer System Using Intel 8257 DMA Controller

8086-based assembly language project that interfaces the Intel 8257
Programmable DMA Controller for high-speed block data transfer.

---

## Source Files

| File | Purpose |
|------|---------|
| `intel8257.inc` | I/O port constants, mode-register bit definitions, and MASM macros |
| `dma_transfer.asm` | Main 8086 assembly program â€“ all four DMA transfer modes |

## Transfer Modes Implemented

| Demo | Channel | Mode | Description |
|------|---------|------|-------------|
| `DEMO_BLOCK` | CH0 | Block | DMA holds bus for entire 256-byte block |
| `DEMO_DEMAND` | CH1 | Demand | Transfers while DREQ active; bus returned on DREQ drop |
| `DEMO_SINGLE` | CH2 | Single-byte | One byte per DMA cycle; 8086 regains bus between bytes |
| `DEMO_MEM2MEM` | CH0â†’CH1 | Memory-to-Memory | On-chip copy using 8257 mode-bit 7 |

## Memory Map

```
Label        Size   Purpose

DEST_BUF     256 B  DMA destination (block / demand / single-byte demos)
SCRATCH_BUF  256 B  Memory-to-memory destination (CH0â†’CH1 demo)
```

Exact DS-relative offsets are assigned by the linker; the buffers are
placed sequentially in the order they appear in the DATA segment.

## How to Assemble (MASM)

```bat
MASM dma_transfer.asm;
LINK dma_transfer.obj;
```

This produces `dma_transfer.exe` which can be loaded into Proteus ISIS
via the 8086 CPU component's program file property.

---

## How to Assemble (TASM)

```bat
TASM dma_transfer.asm
TLINK dma_transfer.obj
```

---

## Proteus Simulation Setup

1. Place an **8086** CPU component and set its program file to
   `dma_transfer.exe` (or `.hex` generated by MASM/TASM).
2. Place an **Intel 8257** component.
3. Connect the 8257 chip-select (`CS`) to the decoded address line
   for base I/O port `00H`.
4. Connect `DREQ0`â€“`DREQ3` to the peripheral data source.
5. Connect `DACK0`â€“`DACK3` and `TC` back to the peripherals.
6. Run simulation and observe the DMA transfer on the data bus.
